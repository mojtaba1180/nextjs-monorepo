# 🚀 اضافه کردن کامپوننت جدید

این prompt برای اضافه کردن کامپوننت جدید به سیستم داکیومنت‌اسیون استفاده می‌شه.

## 📋 دستورالعمل

```
من می‌خوام یک کامپوننت جدید به سیستم داکیومنت‌اسیون اضافه کنم. لطفا مراحل زیر رو به ترتیب انجام بده:

1. **تحلیل کامپوننت**: کامپوننت رو بررسی کن و مشخصاتش رو تعیین کن
2. **ایجاد فایل‌های مورد نیاز**: فایل‌های registry و demo رو بساز
3. **آپدیت registry**: کامپوننت رو به registry اضافه کن
4. **ایجاد داکیومنت**: فایل MDX داکیومنت‌اسیون رو بساز
5. **Build registry**: اسکریپت build رو اجرا کن
6. **تست**: مطمئن شو همه چیز درست کار می‌کنه
```

## 📁 ساختار فایل‌ها

```
src/registry/
├── custom-ui/
│   ├── [component-name]/
│   │   ├── [component-name].tsx      # فایل demo اصلی
│   │   └── [component-name]-variants.tsx  # فایل demo variants
│   ├── index.ts                      # تنظیمات registry
│   └── __index__.ts                  # Auto-generated index

content/docs/custom-ui/
└── [component-name].mdx              # داکیومنت کامپوننت
```

## 📝 قوانین مهم

1. **نام‌گذاری**: از kebab-case برای فایل‌ها و camelCase برای کامپوننت‌ها استفاده کن
2. **Import Path**: همیشه از `@workspace/custom-ui` import کن
3. **فولدرسازی**: هر کامپوننت باید فولدر جداگانه داشته باشه:
   - `src/registry/custom-ui/[component-name]/`
   - فایل اصلی: `[component-name].tsx`
   - فایل variants: `[component-name]-variants.tsx`
4. **Demo Structure**: حتماً 5 مثال مختلف نشون بده:
   - Basic Usage (استفاده پایه)
   - Variants (variant های مختلف)
   - Sizes (سایزهای مختلف)
   - States (حالت‌های مختلف مثل disabled, loading)
   - Interactive Example (مثال تعاملی با state)
5. **Documentation**: داکیومنت کامل با مثال‌ها، API reference و usage بنویس
6. **Registry**: کامپوننت و variants رو به registry اضافه کن
7. **Build**: حتماً اسکریپت build رو اجرا کن
8. **Demo Content**: هر مثال باید محتوای واقعی و قابل نمایش داشته باشه
