## 📋 دستورالعمل کلی

```
من می‌خوام یک کامپوننت جدید به سیستم داکیومنت‌اسیون اضافه کنم. لطفا مراحل زیر رو به ترتیب انجام بده:

1. **تحلیل کامپوننت**: کامپوننت رو بررسی کن و مشخصاتش رو تعیین کن
2. **ایجاد فایل‌های مورد نیاز**: فایل‌های registry و demo رو بساز
3. **آپدیت registry**: کامپوننت رو به registry اضافه کن
4. **ایجاد داکیومنت**: فایل MDX داکیومنت‌اسیون رو بساز
5. **Build registry**: اسکریپت build رو اجرا کن
6. **تست**: مطمئن شو همه چیز درست کار می‌کنه
```

## 📁 ساختار فایل‌ها

```
src/registry/
├── custom-ui/
│   └── [component-name].tsx          # فایل demo کامپوننت
├── index.ts                          # تنظیمات registry
└── __index__.ts                      # Auto-generated index

content/docs/custom-ui/
└── [component-name].mdx              # داکیومنت کامپوننت
```

## 🔧 مراحل پیاده‌سازی

### مرحله 1: ایجاد فایل Demo

```typescript
// src/registry/custom-ui/[component-name].tsx
"use client"

import { [ComponentName] } from "@workspace/custom-ui"

export function [ComponentName]Demo() {
  return (
    <div className="flex flex-col gap-4">
      {/* مثال‌های مختلف کامپوننت */}
    </div>
  )
}
```

### مرحله 2: آپدیت Registry Config

```typescript
// src/registry/index.ts
export const registry = {
  items: [
    // کامپوننت‌های موجود...
    {
      name: "[component-name]",
      description: "توضیح کوتاه کامپوننت",
      type: "registry:component",
      registryDependencies: undefined,
      files: [
        {
          path: "custom-ui/[component-name].tsx",
          type: "registry:component",
          target: "components/[component-name].tsx"
        }
      ],
      categories: ["components"],
      meta: {
        title: "[ComponentName]",
        description: "توضیح کامل کامپوننت"
      }
    }
  ]
}
```

## 📝 قوانین مهم

1. **نام‌گذاری**: از kebab-case برای فایل‌ها و camelCase برای کامپوننت‌ها استفاده کن
2. **Import Path**: همیشه از `@workspace/custom-ui` import کن
3. **Demo Structure**: حداقل 3-4 مثال مختلف از کامپوننت نشون بده
4. **Documentation**: داکیومنت کامل با مثال‌ها، API reference و usage بنویس
5. **Registry**: کامپوننت رو به registry اضافه کن
6. **Build**: حتماً اسکریپت build رو اجرا کن
7. **Installation**: از بخش Installation استفاده نکن، فقط Usage با import statement بذار

## 🎯 مثال کامل

```
کامپوننت: CustomInput
- فایل: custom-input.tsx
- Demo: CustomInputDemo
- Registry: custom-input
- Documentation: custom-input.mdx
```
